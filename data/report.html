Method & caveats

  • Evaluators. All are regex/Python — no LLM and no network at inference. The optional agent_os.prompt_injection_embedding kNN signal is excluded: it is not shipped in the wheel and would require an external embedding model (excluded per the project's no-third-party-model constraint).
  • Ruleset. pi_regex_* use AGT's documented default SAMPLE ruleset. The shipped sample YAML (prompt-injection-safety.yaml) is rule-for-rule identical to these defaults, so a separate YAML row would duplicate.
  • Category note. The real dataset is more granular than the dataset card's 12 categories — the combined splits contain many fine-grained labels (e.g. adversarial, encoding, agent_manipulation, context_confusion).
  • Interpretation. AGT's own README argues input-layer injection detection is a weak, probabilistic surface (cites ~100% ASR studies) and positions AGT around deterministic tool-call policy enforcement. Read these detector numbers as "how good is the sample input detector", not a verdict on AGT as a whole.
