02 — Request anatomy

How a request
travels.

Four checks stand between your application and inference. Watch them fire in order.

TRACE · POST /v1/chat/completionst=0.000s
Origin

Your application

Authenticates to EntraID and receives a JWT scoped to BAI-GATEWAY. Sends the request with an Authorization header.

Control plane

AI Gateway

Validates the token and roles, applies policy, then routes.

ENTRAID AUTH — validate JWT + roles
RATE LIMIT — tenant tier allocation
GUARDRAILS — content-safety inspection
TELEMETRY — span + usage emitted
Destination

AWS Bedrock

The request lands on the resolved model backend in the active region. The response returns through the same plane — inspected and metered on the way out.

TOKEN FLOW · app → EntraID → JWT(aud=BAI-GATEWAY) → Authorization header → gateway validates JWT + roles → routes to model backend

04 — Boarding manifest

Have these
before you board.

Five items, checked at the gate. The onboarding form asks for each one — bring them and the whole thing takes three minutes.

REQ/01

ITSI

An IT Service Instance registered in ServiceFirst for your application — one per environment.

REQ/02

Entra App SPN

An Entra App Registration giving your workload a service identity. No SPN, no plane.

REQ/03

GenAI CoE approval

A JIRA reference from the GenAI Centre of Excellence confirming pilot approval, with Champion sign-off via email.

REQ/04

Models required

The model families your application needs — this determines which Models.<Family>.Invoke roles are assigned.

REQ/05

GenAI CoE sign-off

CSO email approval for the requested families — required by the Entra/IAM team before role assignment.

06 — Scope · Release 1

Read this
before requesting.

FOR

Who boards

  • Applications calling LLMs as part of backend logic
  • Services embedding AI — summarisation, classification, generation
  • Agents and automation pipelines requiring governed LLM access
  • Production workloads with an ITSI, owning team and service identity
  • Machine-to-machine only — not interactive chat for individuals

SHIPS

What you get

  • OpenAI-compatible API — /v1/chat/completions, /v1/models
  • Access to 10+ leading LLMs via AWS Bedrock
  • Identity & access control via EntraID JWT
  • Content-safety guardrails (passive mode)
  • Per-tenant rate limiting with tiered allocation
  • Multi-region active-active with automatic failover

HOLDS

Not in release 1

  • Native Bedrock Converse API — OpenAI-compatible only
  • MCP gateway — tool calling via Model Context Protocol
  • Agent gateway — A2A agent-to-agent communication
  • Multi-provider routing — Azure OpenAI, self-hosted models
  • Inline guardrail blocking — passive/observe only
  • Bring your own model / inference / AWS account
